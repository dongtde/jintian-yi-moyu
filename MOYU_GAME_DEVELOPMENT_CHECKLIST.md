# 《今日宜摸鱼》开发计划清单

> 版本：0.1  
> 依据文档：`MOYU_GAME_PROJECT_PLAN.md`  
> 项目状态：当前仓库已恢复 Vue/Vite 游戏壳、核心摸鱼循环、装备/战斗/存档，并完成 `app / game / ui / assets` 分层。  
> 目标：把大项目方案拆成可以逐步执行、验收、排期和协作的开发清单。

## 1. 开发总目标

《今日宜摸鱼》的开发应分成四个大阶段：

1. **MVP 阶段**：先做出可玩的核心闭环，玩家能点击摸鱼、掉落道具、换装、战斗、升级。
2. **Alpha 阶段**：补齐养成深度，加入搭子、技巧、任务、更多 PVE 和基础真人挑战。
3. **Beta 阶段**：加入工会、工会挑战、工会跳战、活动、排行榜和长期留存系统。
4. **Release 阶段**：接入后端、账号、真实 PVP、工会战、商业化、赛季和运营工具。

第一目标不是做完所有系统，而是尽快做出一个可以反复点击并感到成长的版本。

## 2. 优先级定义

| 优先级 | 含义 | 示例 |
| --- | --- | --- |
| P0 | 没有就不是游戏 | 点击摸鱼、掉落、装备、战斗、存档 |
| P1 | 没有会明显单薄 | 搭子、技巧、任务、PVP 镜像、段位 |
| P2 | 增强留存和社交 | 工会、活动、工位布置、图鉴、成就 |
| P3 | 上线和商业化增强 | 实时 PVP、工会战、后端、战令、商店 |

## 28. 风险和应对清单

### 28.1 玩法风险

- [ ] 风险：核心点击很快无聊。
- [ ] 应对：前 10 分钟高频掉落升级，加入老板雷达和高品质动画。
- [ ] 风险：玩法像换皮。
- [ ] 应对：优先做日报炼丹炉、工会跳战、伪装大赛、老板人格。
- [ ] 风险：流派差异不明显。
- [ ] 应对：每个流派配置专属道具、技巧和克制 Boss。

### 28.2 技术风险

- [ ] 风险：战斗逻辑和 UI 耦合。
- [ ] 应对：战斗系统做成纯函数，UI 只消费结果。
- [ ] 风险：存档升级出错。
